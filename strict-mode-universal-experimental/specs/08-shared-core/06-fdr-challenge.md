# 8.6 FDR Challenge

Part of [Shared Core Components](../08-shared-core-components.md).


Runs on `stop`.

This is the most provider-sensitive component and must be phased later than stub/destructive gates.

Inputs needed:

- normalized `turn.assistant_text`
- normalized `turn.assistant_text_bytes`
- normalized `turn.assistant_text_truncated`
- normalized `turn.edit_count`
- normalized `turn.has_fdr_context`
- current FDR artifact state and `scope_digest`

Claude normalizer can source these fields from `transcript_path`. Codex normalizer may source them from hook payload, transcript path, or other verified runtime state. If a provider cannot populate current-turn fields reliably, its normalized event must leave `turn.assistant_text` empty and FDR challenge must skip with `fdr-challenge skipped: no normalized turn text`.

Core `fdr-challenge.sh` must not contain provider-specific transcript parsing. Provider-specific extraction belongs in `lib/normalize-event.sh` and fixture tests.

v0 FDR challenge must not send raw session history, raw transcripts, full prompts, or full source files to the judge. Provider normalizers may read provider transcripts only to extract the fixture-proven current-turn assistant text and must not write raw transcript content or extracted `turn.assistant_text` into trusted state or diagnostic logs, even when protected full-text capture is enabled. Current-turn assistant text sent to the judge is capped at 32768 UTF-8 bytes. If the extracted current-turn assistant text exceeds that cap, the normalizer must build a deterministic bounded excerpt from the first 16365 bytes plus the last 16365 bytes on valid UTF-8 boundaries, insert the fixed 38-byte marker `[strict-mode-truncated-assistant-text]` between them, set `turn.assistant_text_truncated=1`, and set `turn.assistant_text_bytes` to the bounded excerpt byte length; it must not skip merely because the assistant text is long. If no current-turn assistant text is available, `turn.assistant_text` is empty, `turn.assistant_text_bytes=0`, and `turn.assistant_text_truncated=0`. The challenge skips only when extracting current-turn text would require unbounded history retention, ambiguous turn splitting, or full transcript capture.

`turn.has_fdr_context` is context for the semantic judge only. It must not disable stub scan, artifact validation, unresolved blocked Stop scope carry-over, or FDR challenge when the current edited scope is non-empty. Meta-discussion or FDR-process discussion may skip FDR challenge only when the computed edited scope is empty and no unresolved blocked Stop scope exists.

FDR challenge state:

- `fdr-cycles-<provider>-<sid>.jsonl` is append-only session-scoped trusted state and is written only under the session state transaction lock with trusted-state ledger coverage.
- Each JSONL record has exactly these fields and no extras: `schema_version` (`1`), `provider`, `session_key`, `raw_session_hash`, `cwd`, `project_dir`, `turn_marker`, `cycle_index`, `max_cycles`, `scope_digest`, `tool_intent_seq_list`, `tool_seq_list`, `edit_seq_list`, `tool_intent_log_digest`, `tool_log_digest`, `edit_log_digest`, `content_scope_fingerprint_digest`, `artifact_state`, `artifact_hash`, `artifact_verdict`, `finding_count`, `challenge_reason`, `judge_backend`, `judge_model`, `prompt_hash`, `response_hash`, `decision`, `original_challenge_record_hash`, `bypass_marker_hash`, `bypass_consumed_record_hash`, `bypass_ledger_record_hash`, `ts`, `previous_record_hash`, and `record_hash`.
- `scope_digest` is a full lowercase SHA-256 hex digest over canonical JSON containing cwd, project dir, current-turn tool-intent/tool/edit sequence lists, tool-intent/tool/edit log digests, edited/deleted/renamed paths, and `content_scope_fingerprint_digest`. `artifact_hash`, `prompt_hash`, `response_hash`, `original_challenge_record_hash`, `bypass_marker_hash`, `bypass_consumed_record_hash`, `bypass_ledger_record_hash`, `previous_record_hash`, and `record_hash` are full lowercase SHA-256 hex digests over canonical normalized inputs or 64 zeroes where the field contract says the value is not applicable. For the first record in a session, `previous_record_hash` is 64 zeroes. `record_hash` is computed over the canonical record with only `record_hash` set to an empty string.
- `artifact_state` is one of `missing`, `invalid`, `clean`, `findings`, or `incomplete`. For `missing`, `artifact_hash` is 64 zeroes, `artifact_verdict` is an empty string, and `finding_count` is `0`. For `invalid`, `artifact_hash` is SHA-256 over canonical invalid-artifact evidence containing artifact path, stat metadata when available, parse error code, and redacted/truncated source fingerprint; `artifact_verdict` is an empty string and `finding_count` is `0`. For `clean`, `findings`, or `incomplete`, `artifact_hash` is the normalized artifact hash, `artifact_verdict` equals `artifact_state`, and `finding_count` matches the artifact.
- `decision` is one of `skipped-no-turn-text`, `skipped-trivial`, `judge-unknown`, `judge-clean`, `judge-challenge`, `blocked-reused`, or `bypassed`. Missing normalized turn text records `skipped-no-turn-text` and does not run the judge. Trivial diffs record `skipped-trivial` and do not run the judge. Judge execution failure records audited semantic `judge-unknown`; it does not create an FDR challenge block by itself and does not disable artifact validation or any other Stop gate. `judge-challenge` means the judge returned a challenge and Stop emits a quality block for that record. `blocked-reused` means Stop reused an earlier still-binding challenge/max-cycle block without invoking the judge again. `bypassed` means an approved FDR challenge bypass covers the exact same scope/artifact/original blocking cycle tuple and the generic quality-bypass consumption transaction has already succeeded. For decisions that do not invoke a judge, `prompt_hash` and `response_hash` are 64 zeroes and `judge_backend` and `judge_model` are empty strings.
- `challenge_reason` is a closed decision-specific enum. `skipped-no-turn-text` uses `no-normalized-turn-text`; `skipped-trivial` uses `trivial-diff`; `judge-clean` uses `judge-clean`; `judge-challenge` uses `judge-reported-challenge`; `blocked-reused` uses `reused-blocking-challenge`; `bypassed` uses `approved-quality-bypass`; `judge-unknown` uses exactly one of `judge-disabled`, `judge-invocation-unverified`, `judge-state-isolation-failed-until-repair`, `timeout`, `nonzero-exit`, `invalid-output`, `empty-output`, or `parse-failure`.
- For `judge-clean` and `judge-challenge`, `prompt_hash` is SHA-256 over canonical JSON with no extras containing `provider`, `session_key`, `raw_session_hash`, `cwd`, `project_dir`, `scope_digest`, `artifact_state`, `artifact_hash`, `artifact_verdict`, `finding_count`, `judge_backend`, `judge_model`, `prompt_template_hash`, `assistant_text_sha256`, `assistant_text_bytes`, and `assistant_text_truncated`; `assistant_text_sha256` is computed over the exact bounded assistant-text excerpt sent to the judge, `assistant_text_bytes` and `assistant_text_truncated` must equal the normalized `turn` fields for that excerpt, and the raw text is not stored. `response_hash` must equal the verified canonical `strict-judge` response hash. For `judge-unknown`, `prompt_hash` uses the same prompt hash when a prompt was constructed and a real invocation was attempted, otherwise it is 64 zeroes. `judge-unknown.response_hash` equals the canonical unknown response hash when `strict-judge` returned exact JSON; for timeout, nonzero exit, invalid JSON, or empty output it is SHA-256 over canonical redacted failure evidence with no extras containing `provider`, `session_key`, `raw_session_hash`, `cwd`, `project_dir`, `scope_digest`, `artifact_hash`, `judge_backend`, `judge_model`, `reason_code`, `timed_out` (`0` or `1`), `exit_status` (integer or `-1` when unavailable), `stdout_sha256`, `stderr_sha256`, `stdout_bytes_captured`, `stderr_bytes_captured`, `stdout_truncated` (`0` or `1`), and `stderr_truncated` (`0` or `1`). The unknown response or failure-evidence `reason_code` must equal the cycle record's `challenge_reason`. Raw stdout/stderr and raw prompts are not stored in the cycle record.
- `max_cycles` is exactly `2` in v0. `cycle_index` is a non-negative integer. For `judge-challenge`, it is the 1-based count of distinct artifact hashes that produced a challenge for the same unchanged `scope_digest`. For `blocked-reused` and `bypassed`, it equals the original still-binding `judge-challenge` record's `cycle_index`. For `skipped-no-turn-text`, `skipped-trivial`, `judge-unknown`, and `judge-clean`, it is `0` because those decisions do not consume a challenge cycle.
- `original_challenge_record_hash` is 64 zeroes for `judge-challenge` because the original challenge is the current record and its `record_hash` is not known until after canonical hashing. The emitted quality block and `gate_context.fdr_cycle_record_hash` for that `judge-challenge` must use the completed `record_hash`. For `blocked-reused`, `original_challenge_record_hash` equals the reused still-binding `judge-challenge.record_hash`. For `bypassed`, it equals the original blocking `judge-challenge.record_hash` and must also equal the consumed `bypass-log.jsonl` record's `gate_context.fdr_cycle_record_hash`. For `skipped-no-turn-text`, `skipped-trivial`, `judge-unknown`, and `judge-clean`, it is 64 zeroes.
- For a given pair of `scope_digest` and `artifact_hash`, a trusted prior `judge-clean` record is reused as the non-blocking FDR challenge result without invoking the judge again; other Stop gates may still block. A prior `judge-unknown` is not clean evidence. It may suppress duplicate invocations only inside the same Stop transaction, or across Stop retries only when its `challenge_reason` is deterministic for the current protected config/fixture state (`judge-disabled`, `judge-invocation-unverified`, or `judge-state-isolation-failed-until-repair`). Transient timeout, nonzero-exit, invalid-output, empty-output, or parse-failure unknown records are retried on a later Stop for the same pair. For a given `scope_digest`, `max_cycles` is `2` and counts distinct artifact hashes challenged for that unchanged edit scope. If a successful judge returns a challenge, Stop records `decision="judge-challenge"` and blocks. A later Stop for the same `scope_digest` may invoke the judge again only after the artifact hash changes and the scope has not already reached `max_cycles`. If the same `scope_digest` plus `artifact_hash` repeats, Stop reuses the original still-binding `judge-challenge` record as the quality block subject and must not invoke the judge again. If the same `scope_digest` reaches `max_cycles`, Stop reuses the last blocking challenge or approved bypass record until the tool-intent/tool/edit sequence lists, log digests, or content scope fingerprint changes.
- A reused FDR challenge block must keep `gate_context.fdr_cycle_record_hash` equal to the original still-binding `judge-challenge` record hash, not a newly appended diagnostic `blocked-reused` record. If implementation appends a `blocked-reused` record for audit, that record is diagnostic for cycle history only, must set `original_challenge_record_hash` to the reused `judge-challenge.record_hash`, and must not change the pending bypass hash, approved bypass match, block reason hash, or max-cycle binding. This prevents repeated Stop attempts from invalidating the exact approval phrase already shown to the user.
- A quality bypass for an FDR challenge must bind to the same `scope_digest`, artifact hash, current-turn tool-intent/tool/edit seq lists, tool-intent/tool/edit log digests, content scope fingerprint digest, block reason hash, provider, session key, raw session hash, cwd, project dir, and original blocking `fdr_cycle_record_hash`. A later tool-intent/tool/edit record, artifact change, or new judge-challenge record for a changed scope requires a new challenge state record and a new bypass hash.
- `decision="bypassed"` may be appended only after the matching `bypass-*` marker has been atomically renamed to its consumed tombstone and the consumed audit plus trusted-state ledger records are committed. For `bypassed`, `original_challenge_record_hash` equals the original blocking `judge-challenge.record_hash`, `bypass_marker_hash` equals the consumed marker's original `marker_hash`, `bypass_consumed_record_hash` equals the matching `bypass-log.jsonl` consumed record hash, and `bypass_ledger_record_hash` equals the trusted-state ledger record hash for the consumed tombstone mutation. For all other decisions these three bypass hash fields are 64 zeroes. If bypass verification or consumption fails, the original FDR challenge quality block remains in aggregation and no trusted `bypassed` cycle record is appended.
- Corrupt, missing, non-monotonic, cross-provider, cross-session, or hash-chain-broken `fdr-cycles-*` records make the challenge state untrusted. Untrusted challenge state does not by itself fail closed; it disables reuse/bypass of prior challenge decisions and forces a fresh judge attempt when current-turn fields are available, while other Stop gates keep their normal fail-closed behavior.
